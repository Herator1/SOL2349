Implants are the core augmentation system of _SOL 2349_. They are **active technological systems** integrated into the body and nervous system, designed to push human performance beyond normal limits for brief, decisive moments.

They are not passive upgrades. Implants must be activated, timed, and managed under pressure. Used well, they create powerful tactical advantages. Used recklessly, they can destabilize the user and cause serious consequences.

## What Implants Do

Implants allow characters to:

- act faster
- improve accuracy
- manipulate systems
- survive extreme conditions
- exceed normal human limits for short periods

Implants are one of the main ways characters define their operational role.

## Core Implant Rules

Implants follow a few universal principles:

- most implants are **active**
- most implants require **1 Action** unless stated otherwise
- some implants can be used as **Reactive Actions**
- some implants function as **Augmented Actions**
- activating implants usually generates **[[Load]]**
- implants are limited by **Mod Space**
- characters may only access implants from their chosen categories

Implants are strongest when used at the right moment, not spammed constantly.

## Implants and Load

Every implant activation normally generates **[[Load]]**.

Load represents short-term strain on the body, nervous system, and implant interface. As long as a character remains within safe limits, implants function normally. If current Load exceeds maximum Load, the character risks **Overload**.

The full rules for:

- current and maximum Load
- repeated-use scaling
- overload checks
- overload effects
- recovery

are covered on [[Load]].

## Implants and the Action Economy

Most implants compete directly with everything else a character can do.

Using an implant usually means not:

- attacking
- moving
- interacting with gear
- taking another important action

Repeated implant use in the same round becomes harder to sustain through rising [[Load]] costs.

See [[Time & Actions]] and [[Load]].

## Implant Categories

Implants are organized into distinct operational categories. These categories define what kinds of systems a character can access.

The main categories are:

- [[Combat Implants]]
- [[Physical Implants]]
- [[Spacefaring Implants]]
- [[Survival Implants]]
- [[Engineering Implants]]
- [[Medicine Implants]]
- [[Operations Implants]]
- [[Science Implants]]
- [[Social Implants]]
- [[Psychology Implants]]

At character creation, a character normally selects **2 implant categories**.

This limit does not usually increase through normal progression.

## Hybrid Implants

Some implants belong to **two categories**.

A hybrid implant:

- counts as both categories
- can only be used if the character has access to **both**
- is often more specialized or flexible than a normal implant

Hybrid implants reward focused builds and strong cross-role specialization.

## Augmented Actions

Some implants allow actions outside the normal action flow. These are called **Augmented Actions**.

This changes **how the implant is paid for in actions**, not when it can be triggered.

An Augmented Action:

- does **not** consume a normal Action
- may be used during the character’s turn or reactively, if its timing allows
- still generates **[[Load]]**
- still follows all stated implant limits

Augmented Actions are never truly free. They trade Action cost for rising system strain.

## Reactive Implant Use

Some implants can be activated in response to an event.

This changes **when the implant can be used**, not whether it generates Load or follows normal limitations.

Examples may include:

- emergency stabilization
- reflex enhancement
- sudden sensory boost
- defensive repositioning support

A reactive implant:

- still needs a valid trigger
- still generates [[Load]]
- still follows timing restrictions
- usually costs **1 Action**, unless stated otherwise

If multiple characters react to the same event, resolve them using the normal combat timing rules.

## Mod Space

Each implant uses **Mod Space**.

Mod Space is the long-term capacity limit for how many implants a character can install. [[Load]] limits short-term use. Mod Space limits long-term configuration.

Together, these mechanics define both:

- how much augmentation a character can carry
- and how hard they can push it in the moment

## Implants vs Genetic Modifications

Implants and [[Genetic Modifications]] are not the same.

**Implants:**

- active
- controllable
- generate [[Load]]
- use Mod Space
- can often be installed, replaced, or removed

**Genetic Modifications:**

- passive
- always active
- generate no Load
- use no Mod Space
- are permanent biological changes

Implants offer control and flexibility. Genetic modifications offer constant adaptation with permanent trade-offs.

## Implant Access and Progression

Implants are not normally improved through rank-ups like skills.

Characters improve implant performance through:

- gaining access to new implant options
- increasing supporting stats such as **Maximum Load**
- increasing **Mod Space**
- specializing more deeply in their chosen categories

Implant progression should feel like expanding technological capability, not stacking endless passive bonuses.

## Design Principles

Implants in _SOL 2349_ are built around a few core ideas:

**Burst performance, not permanent superiority.**  
Implants are strongest in short, decisive windows.

**Trade-offs matter.**  
Every activation competes with actions, generates strain, or carries risk.

**Specialization matters.**  
Characters should not be able to access every implant path at once.

**Technology creates identity.**  
Implants are one of the clearest expressions of what a character actually is in play.
