Characters in _SOL 2349_ improve through **Progress Points (PP)**.

These represent experience gained through:

- missions
- problem-solving
- meaningful decisions under pressure
- survival and adaptation in hostile situations

Progress Points are used to improve:

- **[[Skills]]**
- **[[Attributes]]**
- selected **[[Base Stats]]**
- long-term augmentation capacity

Progression uses the same point-buy logic as [[Character Creation]].

## Advancement Rule

Improving a value costs a number of Progress Points based on the **next level**.

**Standard Cost:**  
**Cost = next level**

This creates progressive scaling:

- early improvements are cheap
- later improvements become significantly more expensive

### Example

To raise a Skill from **2 to 3**, the cost is **3 PP**.  
To raise it from **3 to 4**, the cost is **4 PP**.

## Skill Advancement

Skills use the standard progression rule.

|Skill Level|Total Cost|
|---|---|
|0 → 1|1|
|0 → 2|3|
|0 → 3|6|
|0 → 4|10|
|0 → 5|15|
|0 → 6|21|
|0 → 7|28|

These values show the **total cumulative cost** to reach that level from **0**.

## Attribute Advancement

Attributes improve more slowly than Skills.

**Attribute Cost = 2 × next level**

This keeps attributes important, but prevents characters from raising broad baseline capability too quickly.

## Improving Base Stats

Some [[Base Stats]] can also be improved through PP.

Not all Base Stats should advance equally.

### Health

Health uses the **standard progression**.

Health is a strong survivability stat, but increasing it is less structurally powerful than raising system limits or permanent thresholds.

### Maximum Load

**Maximum Load** uses **double progression**:

**Cost = 2 × next level**

Increasing Maximum Load allows heavier implant use and directly improves augmentation endurance.

### Maximum Wounds

**Maximum Wounds** also use **double progression**:

**Cost = 2 × next level**

Because Wounds define how close a character is to death, this should remain a major investment.

### Mod Space

**Mod Space** uses **double progression**:

**Cost = 2 × next level**

Increasing Mod Space expands long-term implant capacity and has major build impact.

### Guard

**Guard cannot be purchased directly.**

It is a derived stat based on [[Attributes]], equipment, and other situational factors.

## Implants and Progression

Implants are not advanced by leveling up individual implant ranks. Instead, progression affects augmentation through:

- gaining access to new implants
- increasing **Mod Space**
- improving supporting stats such as **Maximum Load**
- investing further in the chosen specialization path

## Implant Category Limit

Characters remain limited to **2 implant categories**.

This does **not** change through normal progression.

That limit is a core part of character identity and prevents unrestricted multi-role augmentation builds.

## Design Intent

Progression in _SOL 2349_ is meant to create:

- steady early growth
- meaningful specialization
- increasing cost for mastery
- hard choices between breadth, survivability, and augmentation

Characters can improve significantly, but never without trade-offs.
