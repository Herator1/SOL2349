# 8. [[Rest & Recovery]]

## Raus

- keine Sonderfälle für zehn verschiedene Recovery-Subtracks
- keine halbe Abstufungsliste für jede denkbare Ruhesituation

## Vereinfachen

### Behalte nur:

#### Short Rest

- restore Guard
- recover 1d6 Health
- reduce Mental Condition by 1
- reset current Load to 0

#### Extended Rest

- restore all Health
- remove all Mental Conditions
- reset current Load to 0
- Wounds do not heal automatically

Das ist gut. Das würde ich **nicht** weiter aufblasen.

## Behalten

- current form ist schon ziemlich lean

# 9. [[Load]]

## Raus

- keine weiteren Sondermodifikatoren auf Load jenseits von:
    - implant use
    - repeated implant use
    - some stims

## Vereinfachen

### Buff-/Penalty-Policy rund um Load

Nicht noch zusätzliche kleine Regeln wie:

- high Load directly causes -1 to x
- Load thresholds give passive penalties
- soft caps / yellow zone / red zone

Das würde ich **nicht** einführen.

## Behalten

- current Load
- max Load
- overload only when exceeding
- action still resolves
- reset on rest

Das ist sehr gut und spielbar.

# 10. [[Mental Strain & Psychological Pressure]]

## Raus

- keine zusätzliche numerische Stress-Ressource
- keine zweite parallele mentale Track-Mechanik

## Vereinfachen

### Mental system klein halten

Nur:

- Unsettled
- Shaken
- Strained
- Breaking Point

Nicht mehr.

### Trigger

Nicht versuchen, jede Quelle exakt auszudifferenzieren.  
Default:

- most major shocks add +1 level
- severe events may add more

## Behalten

- Conditions statt Punktepool
- Breaking Point with immediate loss of control
- recovery via rest and support

# 13. [[Implant Overview]] — vor den Implantlisten ganz wichtig

## Raus bzw. als Designregel festlegen

**Implants sollen nicht primär numeric buff engines werden.**

Also möglichst vermeiden:

- +2 to hit
- +2 to checks
- crit range +1
- passive stackable combat bonuses
- many 2-turn / 3-turn microbuffs

## Stattdessen bevorzugen

Implants sollten vor allem:

- neue taktische Optionen öffnen
- Timing verändern
- Positioning verändern
- Reaktionen erlauben
- Informationsvorteile geben
- Umgebungen beherrschbar machen
- Rollenidentität schärfen

### Gute Implant-Effekte:

- move differently
- react to triggers
- perform task remotely
- survive vacuum longer
- bypass access restrictions
- reduce one category of penalty
- perform emergency intervention

### Schlechte Implant-Effekte für dein Ziel:

- “+2 on all ranged attacks for 3 turns”
- “gain Edge, +2 damage, and crit on 19–20”
- “stack three small buffs before attacking”

# 14. Globale Systemregel, die du festlegen solltest

## **Buff Policy**

Ich würde systemweit festschreiben:

### Es gibt nur 3 primäre positive Effektformen:

- **Tactical Edge**
- **+1 Action**
- **+1 damage die** _(selten)_

Und nur 3 primäre negative Kurzzeiteffekte:

- **Setback**
- **lose 1 Action next turn**
- **+1 Mental Condition** / existing condition worsens

Alles andere nur, wenn es wirklich ikonisch und selten ist.

Das wäre wahrscheinlich der wichtigste Hebel im ganzen System.
